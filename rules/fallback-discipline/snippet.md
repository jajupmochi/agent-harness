- Fallback discipline: judge every `pass` / swallowed exception / broad `except` / default-substitution by scenario. **Production** — allowed (often required) to avoid a user-facing crash, but it MUST log the trigger, input, why the primary failed, and what it did. **Test/dev** — presumed guilty; if it could hide any bug, fix it on the spot or raise loudly, never silently pass. The dividing question: "if this fallback fires, does someone find out?" If no one does, it's wrong in both.

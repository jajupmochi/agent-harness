- Root cause before fix: before fixing any bug, reproduce + locate it (`file:line`), read the failing line's history (`git log -L` / blame), and compare against the baseline branch to classify **regression vs pre-existing latent fragility** and answer "why now?". Trace the bad value to its origin, then fix at the correct layer (guard / revert introducing commit / fix the data-caller) and record the finding in the commit. Never patch-first; a guard that hides an upstream regression is the wrong layer.
